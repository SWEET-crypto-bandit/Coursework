# Coursework — браузерная игра (HTML/JS)

Небольшая браузерная игра на чистом **HTML/CSS/JavaScript** с локациями, диалогами, инвентарём, боями и сохранениями в `localStorage`.

## Возможности

- **Карта/локации**: переходы между локациями через диалоговые окна.
- **Диалоги**: `choiceDialog2/3/5`, текстовые сцены, видео-сцены.
- **Инвентарь и торговля**: предметы, покупка/продажа, интерфейс инвентаря.
- **Бои (PvP/PvE)**: пошаговые действия (атака/блок/хил), разные противники.
- **Сохранение**: сохранение/загрузка игрока в `localStorage`.
- **Бизнес у директора**:
  - покупка бизнеса за **5000 монет**;
  - после покупки бизнес **копит +50 монет за каждый ход**;
  - можно **забрать накопленные деньги** отдельной кнопкой в кабинете директора.

## Быстрый старт

### Вариант 1 — просто открыть HTML

1. Открой `start.html` в браузере.
2. Нажми **Играть**.

> Примечание: из‑за особенностей браузера некоторые вещи (например, автоплей аудио/видео) могут требовать взаимодействия (клик/кнопка/пробел).

### Вариант 2 — запуск через локальный сервер (рекомендуется)

Это помогает избежать ограничений браузера при работе с файлами.

**Node.js (http-server):**

```bash
npx http-server .
```

**Python:**

```bash
python -m http.server 5173
```

После этого открой в браузере страницу, которую покажет сервер, и перейди к `start.html`.

## Управление и интерфейс

- Основное управление — через **кнопки диалогов** (выбор действий).
- Вверху отображаются основные статы: **HP**, **деньги**, **еда**, **радиация**.
- Есть кнопка **инвентаря** (использование/продажа предметов зависит от текущей локации).

## Как работает “ход”

В игре есть понятие “хода”: когда ты переходишь между локациями/сценами, вызывается `applyTurnCosts()` (см. `move.js`).

За ход:

- тратится еда/здоровье по правилам выживания;
- в пустоши отдельно учитывается радиация;
- **если куплен бизнес директора — накапливается +50 монет в баланс бизнеса**.

## Структура проекта

- `start.html` — точка входа, подключение скриптов.
- `player.js` — глобальный объект `Player` (статы, инвентарь, бизнес).
- `types.d.ts` — типы (используются с `// @ts-check` в JS файлах).
- `templates.js` — UI диалогов (`creayDialog`, `choiceDialog*`) и вспомогательные функции.
- `interface.js` — обновление верхнего HUD (`upInterface`) и отрисовка инвентаря.
- `invertari.js` — логика предметов (использование/удаление/добавление).
- `move.js` — стоимость хода (`applyTurnCosts`) и связанные эффекты.
- `pvp.js` — логика боёв.
- `map/` — локации/сцены:
  - `map/map.js` — основные переходы по локациям
  - `map/shop/`, `map/snack bar/`, `map/hospital/` и т.д.
  - `map/director/director.js` — кабинет директора и бизнес
- `music/` — видео/аудио ассеты проекта.

## Сохранения

Сохранение игрока хранится в `localStorage` (ключ: `player_save_v1`).

- сохранить/загрузить/удалить можно из меню больницы (см. `map/map.js` → `hospital()` и `save.js`).

## Разработка

### Требования

- Любой современный браузер (Chrome/Edge/Firefox).
- Опционально: локальный сервер (Node/Python).

### Типы и проверка

Часть файлов использует `// @ts-check`. Типы описаны в `types.d.ts`.

Если добавляешь новые глобальные функции (через `<script>`), удобно дополнять `types.d.ts` блоком `declare global { ... }`, чтобы не ловить ошибки “Cannot find name …”.
